    Preparazione e avvio di Atmosphère#

    Atmosphère permette a tutti gli effetti di avere pieno controllo sulla console, facendo in modo di avviare tutti gli homebrew desiderati.


    INSTALLAZIONE PULITA

    NEL CASO IN CUI SI STIA EFFETTUANDO UN'INSTALLAZIONE PULITA DI ATMOSPHERE, BISOGNERÀ CANCELLARE (MAGARI FACENDO PRIMA UNA COPIA DI BACKUP PER SICUREZZA) LE SEGUENTI CARTELLE E IL FILE DALLA MICROSD:

    ● <microSD>/atmosphere/
    ● <microSD>/bootloader/hekate_ipl.ini
    ● <microSD>/sept/



    !!! IN TAL MODO QUALSIASI CHEAT O MOD COPIATO IN PRECEDENZA NELLA MICROSD VERRÀ CANCELLATO !!!
    NESSUN GIOCO O SALVATAGGIO VERRÀ CANCELLATO.

    DOPO AVER FATTO CIÒ, SI PUÒ CONTINUARE CON LA GUIDA.


    1. Prerequisiti#

    • L'ultima release di Atmosphère (atmosphere-x.xx.x-master-xxxxxxxx+hbl-x.x.x+hbmenu-x.x.x.zip e fusee.bin);


    2. Preparazione dei file#

    2.1 Atmosphère#

    Estrarre direttamente sulla microSD i file scaricati precedentemente in modo da ottenere file e cartelle come nella figura che segue.

    NOTA BENE: La cartella emuMMC sarà presente solo nel caso in cui si sia seguita questa guida per la creazione della emuNAND in precedenza. In caso contrario, la cartella non sarà presente.

    2.2 Modalità incognito#

    Warning

    ATTIVANDO QUESTA MODALITÀ NON SARETE PIÙ IN GRADO DI CONNETTERVI AI SERVER UFFICIALI.

    Con questa modalità si farà in modo di cancellare virtualmente la partizione PRODINFO, ovvero quella responsabile del riconoscimento online della vostra console, in modo tale da evitare ogni preoccupazione riguardante eventuali ban.

    Per attivare la modalità incognito bisogna:

    1. Recarsi nella cartella: /atmosphere/config_template/;
    2. Copiare il file: exosphere.ini;
    3. Recarsi nella directory principale della microSD;
    4. Incollare il file copiato in precedenza;
    5. Aprire il file exosphere.ini e impostare a 1 i valori seguenti come mostrato in figura; quindi:
      • blank_prodinfo_sysmmc = 1;
      • blank_prodinfo_emummc = 1.

    MI RACCOMANDO!

    È DI VITALE IMPORTANZA CHE QUESTO FILE SIA SEMPRE PRESENTE NELLA MICROSD.

    !!! SE DOVESSE NON ESSERE PIÙ PRESENTE, UNA VOLTA CONNESSI, SARETE BANNATI. !!!

    SI RACCOMANDA ALLORA ANCHE DI TENERE LA MODALITÀ AEREO SEMPRE ATTIVA.

    2.3 File hosts#

    Come ulteriore protezione, dalla versione 0.18.0 è possibile reindirizzare le richieste DNS attraverso l'utilizzo di un file hosts. Nel caso in cui questo file non esista, verrà creato in automatico al primo avvio di Atmosphère.

    Per avere più informazioni: FONTE


    Per abilitare e ampliare questa funzionalità bisogna:

    1. Creare la cartella: /atmosphere/hosts/;
    2. Recarsi nella cartella appena creata;
    3. Creare un nuovo file di testo: default.txt;
    4. Incollarvi il seguente contenuto:

    3. Modalità RCM & fusee#

    Bisogna a questo punto inserire la microSD nella console e metterla in modalità RCM, un particolare stato di funzionamento che permette di inviare (iniettare) i file necessari ad avviare la console.

    Per mettere la console in questa particolare modalità bisogna:

    1. Assicurarsi di avere la console completamente spenta;
    2. Inserire il jig nel binario del joycon destro della console;
    3. Tenendo premuto il tasto Volume +, preme una volta il tasto di accensione.

    La console sarà adesso in modalità RCM e non darà alcun segno di vita, ed è normale.

    4. Avvio di Atmosphère#

    Se tutto è andato a buon fine, la console dovrebbe avviarsi con l'immagine di avvio di Atmosphère:

    CAMBIA TEMA!

    NEL CASO IN CUI SI SIA AVVIATA L'EMUNAND, APPLICATE UN TEMA DIVERSO ALLA SYSNAND IN MODO TALE CHE SIA DI FACILE RICONOSCIMENTO.
    PER ESEMPIO: SYSNAND = TEMA CHIARO, EMUNAND = TEMA SCURO.

    Una volta avviata la console, per verificare che Atmosphère sia stato avviato correttamente, basterà andare in Impostazioni di sistema, Sistema e verificare che vi sia la versione di Atmosphère che si sta utilizzando accanto alla Versione di sistema attuale (la E indica l'utilizzo della emuNAND, mentre la S indica l'utilizzo della sysNAND).
